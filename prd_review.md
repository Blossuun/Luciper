# PRD 아키텍처 리뷰 — 철권 특화 로컬 STT

## 총평

전체적으로 **문제 정의와 레이어 분리가 명확하고, "모델이 아닌 후처리와 UX로 차별화한다"는 판단**이 정확하다. 세션 종료 시점에만 학습을 제안하는 UX 철학도 실무적으로 옳은 선택이다. 그러나 **Tauri + Python 이중 런타임 조합의 IPC 복잡도**, **실시간 partial/final 자막 안정화의 엔지니어링 난이도**, **공유 용어집 서버 운영의 비용·보안 부담**이 MVP 단계에서 심각한 병목이 될 수 있다. 특히 사용자 base가 0에서 시작하는 로컬 배포 제품에서 **Trust Scoring 기반 공유 용어집은 네트워크 효과가 발현되기 전까지 죽은 기능**이다. GPU 요구사양, 크로스플랫폼 전략, 모델 배포 방식 등 **운영 레벨의 핵심 제약 조건이 문서에 빠져 있어** 구현 시 즉시 의사결정 공백이 발생한다. 다른 게임으로의 확장을 고려하면 "게임 프로필" 개념의 아키텍처 레벨 추상화가 초기부터 필요하다.

---

## 잘한 점

- **"모델 재학습 < 후처리 + 용어집 루프"라는 전략적 판단**이 정확함. Whisper 계열 모델의 도메인 적응은 fine-tuning보다 prompt engineering + 후처리가 비용 대비 효과적이라는 업계 합의와 일치
- **세션 종료 시점 일괄 학습 제안 UX** — 수정 중 방해 금지 원칙이 실제 자막 편집 워크플로우와 잘 맞음. 사용자 저항 최소화에 유효한 설계
- **적용 우선순위 (Local > Shared > Built-in)**가 명확하여 오프라인 퍼스트 원칙과 일관성 유지

---

## 문제점 / 리스크

- **Tauri + Python 이중 런타임**: Tauri(Rust) ↔ Python(faster-whisper) 간 IPC 설계가 PRD에 빠져 있음. 이것이 아키텍처의 **가장 큰 기술 리스크**. sidecar 프로세스 관리, 크래시 복구, 로깅 통합 모두 비자명한 문제. Electron + child_process가 현실적으로 훨씬 빠르게 돌아감
- **GPU 요구사양 미정의**: faster-whisper의 `small` 모델도 VRAM 2GB+ 필요. CPU fallback 시 실시간 추론이 불가능할 수 있으며, 이 경우 "실시간 자막"이라는 핵심 UX가 깨짐. **최소 하드웨어 요구사항과 모델 크기별 지연 시간 벤치마크**가 반드시 문서에 있어야 함
- **공유 용어집 보안 설계가 선언적**: "교차 검증", "Trust Scoring"이라고만 적혀 있고 구체적 알고리즘 없음. **Sybil attack(다중 계정)에 대한 방어선이 전무**하며, 초기 사용자 수 부족 시 교차 검증 자체가 작동하지 않음. 사용자 인증 메커니즘(OAuth, device fingerprint 등)도 미정의
- **타임라인형 자막 편집기의 개발 비용 과소평가**: 타임코드 기반 편집 UI를 자체 구현하는 것은 그 자체로 중규모 프로젝트. waveform 표시, 구간 드래그, 시간 정렬 등을 포함하면 **V1 일정을 최소 2배 이상 늘림**
- **다중 게임 확장을 위한 아키텍처 추상화 부재**: 현재 구조가 "철권"에 하드코딩될 위험. Post-processing 규칙, 용어집 스키마, UI 프리셋 등이 게임별로 분리 가능한 **프로필/플러그인 구조**로 설계되어야 수평 확장이 가능

---

## MVP 축소안

- **Layer 6(Shared Lexicon Sync) 전체 제거**: V1은 순수 로컬 제품으로 출시. 공유 용어집은 V2에서 "용어집 JSON export/import" → "GitHub 기반 수동 공유" → "서버 기반 자동 동기화" 순으로 점진 도입
- **타임라인 편집기 → 단순 리스트 편집기로 대체**: 타임코드 표시 + 텍스트 인라인 편집만 지원. waveform/드래그는 V1.5 이후. 기존 오픈소스(예: subtitle-editor 웹 컴포넌트)를 래핑하는 것도 고려
- **Tauri 대신 Electron 또는 순수 Web UI(localhost) 시작**: Python FastAPI 서버를 localhost에 띄우고 브라우저에서 접근하는 구조가 MVP로는 가장 빠름. 데스크톱 앱 패키징은 사용자 피드백 확인 후 전환

---

## 최종 권고

V1의 핵심은 **"로컬 faster-whisper 추론 + 철권 후처리 + 편집 UI + 세션 종료 학습"**이며, 이것만으로도 충분한 가치 검증이 가능하다. **공유 용어집과 Tauri 패키징은 V1의 스코프에서 명확히 제외**하고, Python + Web UI(localhost)로 빠르게 프로토타입을 만들어 **실제 철권 커뮤니티에서 후처리 정확도와 편집 UX를 검증**하는 것이 우선이다. 다중 게임 확장을 고려한다면, 용어집 스키마에 `game_id` 필드를 초기부터 포함하고 후처리 규칙을 게임 프로필 단위로 분리하는 최소한의 추상화만 V1에 심어두면 된다. GPU 요구사양과 모델 크기별 성능 벤치마크 표는 **README와 PRD 양쪽에 반드시 포함**해야 한다.
